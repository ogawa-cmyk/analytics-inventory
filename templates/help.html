このツールについて

Analytics Inventory は、複数のGmailアカウントが所有する GA4プロパティ・GTMコンテナ・Search Consoleサイト を一覧管理し、健全性スコアリング・差分検知・AI分析を行うインベントリ管理ツールです。

  • GA4 / GTM / SC のAPIを使い、3日ごとに自動でデータ収集（タスクスケジューラ）
  • 各プロパティ・コンテナ・サイトを 100点満点でスコアリング、アラートを自動検出
  • Claude APIに分析を投げ、課題リストと優先度付きアクションプランを生成
  • 各アクションには外部AIに作業を委任するためのプロンプトも同時生成
  • 更新後の新規アラート発生プロパティを 自動診断 で即AI分析

用語集

GA4関連

プロパティ (Property)
GA4でデータを収集・保管する単位。1サイト=1プロパティが基本。property_idで識別。
データストリーム (Data Stream)
ウェブ／iOSアプリ／Androidアプリのいずれかからのデータ収集口。1プロパティに複数設定可能。Measurement ID（G-XXXXX）はWebストリーム単位で発行。
キーイベント (Key Event)
旧称「コンバージョン」。ビジネス価値の高いイベントとしてマークしたもの。GA4管理画面では2024年10月から「キーイベント」に名称変更。
カスタムディメンション (Custom Dimension)
GA4標準にない属性を計測するための拡張枠。スコープがEvent / User / Item / Sessionの4種類。プロパティあたり50件が上限。
カスタム指標 (Custom Metric)
数値型のカスタム計測。スクロール時間や購入額など。プロパティあたり50件が上限。
Measurement ID (MID)
G-XXXXXXXXXX形式の識別子。GA4でデータを受け取る側のID。GTMのGA4設定タグやgtag.jsにこのIDを指定して計測が成立。
accessBindings
GA4 Admin APIで取得できる権限情報。Administrator権限を持つプロパティでのみ全件取得可能（Editor以下では空配列）。
eコマース計測
purchase / add_to_cart / begin_checkout / view_item等のイベントが直近30日で1回以上発生していれば「eコマースあり」と判定。

GTM関連

コンテナ (Container)
GTMでタグ・トリガー・変数をまとめる単位。基本は1サイト=1コンテナ。container_id（数値）とPublic ID（GTM-XXXXXXX）の2識別子。
公開バージョン (Live Version)
現在実環境で配信されているコンテナのスナップショット。「公開」のたびに新バージョンIDが発行。本ツールは公開バージョンの内容のみを収集。
タグ (Tag)
計測コードを発火させる単位。GA4設定タグ（gaawc）、GA4イベントタグ（gaawe）、カスタムHTMLタグ（html）、Yahoo!広告タグなど。
トリガー (Trigger)
タグを「いつ・どの条件で発火させるか」を定義する単位。ページビュー、クリック、フォーム送信、カスタムイベントなど。
変数 (Variable)
タグやトリガーで使う動的な値の定義。データレイヤー変数、URL、Cookie値、定数など。
レガシーUA (Universal Analytics) タグ
2023年7月計測終了のGoogle Analytics 3世代タグ。タグタイプua。即削除推奨。
paused フラグ
タグを一時停止状態にするフラグ。本番にpausedタグが多いとガバナンスが乱れているサイン。

Search Console関連

サイト（プロパティ）
Search Consoleで監視する単位。2種類: URLプレフィックスプロパティ（https://example.com/）と ドメインプロパティ（sc-domain:example.com、サブドメイン全部対象）。
権限レベル (Permission Level)
SC固有の権限種別: siteOwner（所有者）/ siteFullUser（フルユーザー）/ siteRestrictedUser（制限付き）/ siteUnverifiedUser（未確認）。
Click / Impression / CTR / 平均掲載順位
Search Consoleの4基本指標。Click=実際にクリックされた数、Impression=検索結果に表示された回数、CTR=Click÷Impression、平均掲載順位=検索結果で表示された平均的な順位。
クエリ (Query)
ユーザーがGoogle検索で入力した語句。プライバシー上一部マスクされる。
サイトマップ (Sitemap)
サイトに含まれるURL一覧をXMLで提供したファイル。クロール最適化に重要。Search Consoleで送信状況・取得日時・エラー数を確認可能。
本ツールの集計期間
直近28日のClick/Impression/CTR/平均掲載順位を集計（SC APIの仕様で3日の遅延あり）。クエリ・ページの上位100件を保存。
GA4自動紐付け
GA4プロパティのデータストリームURLのドメインと、SCサイトのドメインを照合して自動連携検出。一致するGA4プロパティが見つかれば「GA4連携あり」表示。

本ツール独自の用語

ヘルススコア (Health Score)
各プロパティ・コンテナ・サイトを0〜100点で評価した健全性指標。判定の詳細はスコア基準セクション参照。
グレード (A〜F)
A（80〜100）・B（60〜79）・C（40〜59）・D（20〜39）・F（0〜19）の5段階。A〜Bは健全、C以下は改善余地ありの目安。
アラート (Alert)
機械的に検出された問題点。error（重大・即対応）/ warn（注意・要改善）/ info（情報・参考）の3レベル。
スナップショット (Snapshot)
データ収集のたびに保存される時点データ。data/snapshots/に最大20件保持。前回比較で変化検知に使う。
委任プロンプト (Delegate Prompt)
AI分析が出力するアクションに付属する、外部AI（Claude/ChatGPT等）に作業を引き継ぐための完全自立型プロンプト。1,000〜2,000字程度。
自動診断 (Auto Diagnose)
3日ごとのデータ更新後に自動実行される機能。前回スナップショット比で新規アラート発生プロパティを最大15件AI分析し、結果ジョブを生成。
一括AI分析
複数プロパティをまとめてClaude APIに分析させる機能。プロパティ一覧でチェックボックス選択 → 概算確認 → ジョブ起動 → 進捗ポーリング → 結果横並び表示。
紐付き不整合
Measurement IDをキーにGA4プロパティとGTMコンテナを相互紐付け。「GA4にMIDがあるのに受信するGTMが見当たらない」場合は不整合として警告。
ツールタグ
ホームの要対応タイル左上に表示される小さな色付きラベル。📊 GA4（緑）/ 🏷 GTM（紫）/ 🔍 SC（青） の3種。クロス系は両色グラデーション。

データ更新の仕組み

自動更新（3日ごと）

  • Windowsタスクスケジューラの Analytics-Inventory-AutoRefresh が3日ごとに run_indexer.bat を実行
  • すべてのアカウントを巡回し、データを data/inventory.json に保存
  • 収集完了後、auto_diagnose.py が自動的に走り、新規アラート発生プロパティを最大15件AI分析
  • 所要時間: 20〜45分（プロパティ数・SCサイト数に依存）

手動更新

  • ヘッダ右上の「⟳ 更新」ボタンから即実行
  • 進捗パネルが自動表示: 進捗バー / 経過時間 / 残り時間（ETA） / 処理速度 / 現在処理中
  • ステータス表示をクリックで進捗パネルの開閉可
  • 5秒ごとに自動更新、完了で自動リロード

スナップショット保存

収集のたびに data/snapshots/{timestamp}.json に当時の状態を保存。最新20件まで保持。プロパティ詳細での前回比差分表示に使用。

スケジュール変更

PowerShell/cmdで間隔変更可:

schtasks /Delete /TN Analytics-Inventory-AutoRefresh /F
schtasks /Create /TN Analytics-Inventory-AutoRefresh /TR "C:\Users\<ユーザー名>\analytics-inventory\run_indexer.bat" /SC DAILY /MO 3 /ST 03:00 /F

AI分析の仕組みとコスト

使用モデルとコスト目安

モデル用途1分析あたり目安
claude-sonnet-4-6標準分析（既定）$0.08〜0.10 / 約12円
claude-opus-4-8深い分析$0.40〜0.45 / 約65円

分析の3つの実行方法

  1. 個別AI分析: プロパティ詳細・GTMコンテナ詳細の「🤖 AI分析」セクションから1件ずつ実行
  2. 一括AI分析: プロパティ一覧でチェックボックス選択 → 「🤖 一括AI分析」ボタン → モデル選択 → 概算確認 → ジョブ起動
  3. 自動診断: 3日ごとのデータ更新後に新規アラート発生プロパティを最大15件自動分析（または「分析」ドロップダウン→「自動診断を実行」で手動起動）

分析対象として送信される情報

  • プロパティ基本情報（名前・ID・業種・タイムゾーン・通貨・自分のロール）
  • ヘルススコアとグレード
  • 計測状況（直近7日の Sessions / Events / Tracked判定）
  • データストリーム一覧（Measurement ID込み）
  • キーイベント、カスタムDim/Metric一覧
  • eコマース検出状況
  • 直近30日のイベント上位20件（回数・ユニークユーザー数）
  • 紐づくGTMコンテナ

サイト訪問データそのものは送信しません。設定情報と集計済みイベント名・件数のみ。

AI出力の構造

JSON形式で以下を返します:

  • summary: 2〜3行の総合所見
  • issues[]: 課題リスト（最大8件）。severity（high/medium/low）・category・title・description
  • action_plan[]: アクションプラン（最大6件）。priority・title・description・estimated_effort・delegate_to_ai・delegate_prompt

過去履歴の保存

  • GA4: data/ai_runs/{property_id}/{timestamp}.json
  • GTM: data/ai_gtm_runs/{container_id}/{timestamp}.json
  • 一括ジョブ: data/bulk_jobs/{job_id}.json
  • 各プロパティ・コンテナごとに最新10回分を保持

トラブルシューティング

サーバが起動しない / アクセスできない

デスクトップの「GA4 Inventory起動.bat」をダブルクリック、もしくは start_server_silent.bat を実行。8788ポート競合時は stop_server.bat で既存プロセスをkill。

AI分析が失敗する（JSON解析失敗）

Claude APIの稀な空応答が原因のことが多い。2回まで自動リトライしますが、失敗する場合はOpus 4.8に切り替えて再実行を。エラーレスポンスは data/ai_runs/{pid}/{ts}-error.json に保存。

権限不明（my_roles が空）

該当プロパティで自分がAdministrator権限を持っていないと accessBindings.list APIが呼べないGoogleの仕様。実際の権限はGA4管理画面で確認。

GTM 429エラー

Tag Manager APIのレート制限。retry_errors.py でリトライ可能（バックオフ実装済）。

SC収集が0件

Google Cloud Console で Search Console API が有効化されているか確認。詳細はSC一覧ページの未収集メッセージ参照。

自動起動が動かない

Windowsスタートアップフォルダ方式のため、PCログイン時にのみ発火。設定後にログアウト/ログインを済ませる必要があります。

ヘッダのエラー件数をクリック

→ /errors ページに遷移し、収集エラーの詳細（ステージ別・Gmail別・対象別）を確認可。
