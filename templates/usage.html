このツールで何ができるか

複数クライアントの GA4プロパティ・GTMコンテナ・Search Consoleサイト を横断管理し、計測実装の問題を機械検出、AI分析で具体的な改善案を出すまでが目的です。

機械でできること
  • 計測停止・キーイベント未設定・CD過多・GA4↔GTM不整合の自動検出
  • UA系タグ残存・paused率過多・規模過大のGTM評価
  • SC流入0・sitemap未登録・低CTRの検出
  • 過去スナップショットとの差分検出
  • ヘルススコアによる定量評価（GA4/GTM/SC共通の100点満点）
AIに任せられること
  • 業種・URLから推奨キーイベント5件の提案
  • CD命名規約レビューと統合候補の特定
  • eコマース実装チェックリスト作成
  • UAタグ整理・GTMリファクタ計画書の生成
  • 各アクションを外部AIで実行するためのプロンプト生成

毎日のチェック（5分）— ホームの読み方

ホームを開いて上から目を通すだけで、その日に対応すべきことが分かる構成になっています。

1

ファーストビュー（常時表示）の3パートを確認

4列ヘルスサマリ・要対応タイル・クイックアクションがスクロールせずに見えます。

GA4 ヘルス326平均 65点
GTM ヘルス113平均 78点
SC ヘルス45平均 60点
運用状況5認証Gmail

グレード分布バーは A〜F の実件数を比例幅で表示し、各セグメントに%表記。バーが極端に偏っている＝改善余地大。

2

「今日の要対応」タイルで気になる項目をクリック

各タイル左上に 📊 GA4 / 🏷 GTM / 🔍 SC のツールタグ。重大度に応じて赤（error）またはオレンジ（warn）。

📊 GA4 11件のプロパティで計測停止 ›
🏷 GTM 2件のGTMでUAタグ残存
🔍 SC 5件のSCでCTR極低
3

クリックするとモーダルが開き、対象一覧と対策方法をタブ切替で確認

「対象 X件」タブで影響範囲の全件リスト、「📋 対策方法」タブで具体的な手順を見られます。リンクから個別の詳細ページへも飛べます。

プロパティで計測停止 (11件)
対象 11件 📋 対策方法
対象 / グレード / 補足
プロパティ名 / D / sessions7d=0
4

必要ならクイックアクションで一括対応

各ボタンに マウスオーバーで詳細ツールチップ が表示。コストや所要時間が事前に分かります。

▶ データ更新 🤖 自動診断 ⚡ アラート一括AI分析 ⬇ GA4 CSV ⬇ GTM CSV

「▶ データ更新」実行中は ヘッダ右に進捗パネル が表示され、件数・経過時間・残り時間（ETA）・処理速度・現在処理中の対象が見えます。

5

必要に応じて折りたたみセクションを開く

下記4セクションはデフォルト閉。クリックで展開:

  • 📈 最近の変化 — 前回スナップショット比のTOP5
  • 🚨 要対応 TOP10 — GA4 / GTM / SC を内部タブ切替で表示
  • 🤖 AI分析の活用状況 — 自動診断・カバレッジ・最頻出課題TOP5・直近ジョブ
  • ⭐ お気に入り・タグ・MID重複 — 3カード横並び
Tip: ホームは3日に1回のデータ更新の翌日に開くと最も価値があります。スナップショット差分と自動診断の両方が揃った状態を確認できます。

要対応アラートの読み解きと対応フロー

各アラートタイルをクリックすると 「対象」「対策方法」 の2タブモーダルが開きます。

対象タブで見られる情報

  • 対象名（クリックで詳細へ） ・ グレード ・ 補足情報（sessions7d、UA件数、CTRなど）
  • SCのMID重複・GA4↔GTM不整合は特殊レイアウト（重複コンテナ・未受信MIDを直接表示）

対策方法タブで見られる情報

  • この問題について（緑のサマリーボックス）
  • 📝 対策手順（5〜7ステップの番号付きリスト）
  • 🤖 AI委任ヒント（該当する場合、AI分析実行を促す）

対策ガイドが用意されているアラート（13種類）

GA4 (5種)
untracked / no_streams / api_err / no_ke / cd_overflow
GTM (3種)
no_tags / ua_left / no_ga4
SC (3種)
sc_no_clicks / sc_no_sitemap / sc_low_ctr
クロス (2種)
duplicate_mids / mismatch

それぞれに「GA4管理画面の操作手順」「GTM編集手順」「サイト側の調査ステップ」など具体的な行動が記載されているので、見ながら対処できます。

週次の改善フロー（30分）

1

プロパティ詳細ページを開く

GA4プロパティ一覧から対象プロパティの名前をクリック。右サイドにスティッキー目次が表示され、長いページでも目的のセクションへ即移動可。

2

「🤖 AIで分析・アクションプラン生成」を実行

モデル（Sonnet 4.6 / Opus 4.8）を選んで「▶ 分析実行」。30〜90秒で 所見・課題リスト・優先度付きアクションプラン が返ってきます。

3

アクションごとの「🤖 委任可」を選んで外部AIに依頼

各アクションを展開すると、1,000〜2,000字の自立型プロンプトが用意されています。「Claude」「ChatGPT」ボタンで新タブを開き、すぐ作業着手。

4

結果をクライアントに報告／実装に反映

AIが生成した設計書・実装手順を確認後、クライアント担当に共有。プロパティ詳細の「🏷 タグ・メモ」欄に対応状況を記録しておくと履歴管理に便利。

AI分析の活用パターン

① 個別分析（プロパティ/コンテナ詳細から）

1件をじっくり見たい時。所見→課題→アクション→委任プロンプトの流れ。GA4プロパティ・GTMコンテナの両方で対応。

② 一括AI分析（プロパティ一覧から）

10〜20件をまとめて分析したい時。一覧の各行のチェックボックスで選択 → 「🤖 一括AI分析」→ モーダルでモデル選択 → 概算確認 → ジョブ起動。

☑ ☑ ☑ ☑ ☑   ⚠ アラート分を選択 🤖 一括AI分析 (5) ⬇ CSV
推定コスト: $0.46 (約71円) ・ 所要時間: 約5分

③ 自動診断（3日ごとに自動実行）

3日ごとのデータ更新後に自動で動き、新規アラート発生プロパティを最大15件分析。結果はホームに通知。ヘッダの「分析▾」ドロップダウン →「自動診断を実行」で手動起動も可。

④ AI分析カテゴリ集計（ホーム層3）

複数プロパティのAI結果を集計し、最頻出の課題カテゴリTOP5が見えます。「キーイベント未設定が25件」のような全体傾向把握＝社内レポートやセミナー素材に。

コスト感: Sonnet 4.6で1分析あたり約12円、Opus 4.8で約65円。10件の一括で約120円。自動診断1ヶ月分（3日×10回×平均10件）で約1,200円。

クライアント別の管理（タグ・お気に入り・メモ）

お気に入り（★）

プロパティ一覧・GTM一覧・各詳細ページで★クリックで登録。ホームの折りたたみ「⭐ お気に入り」セクションに固定表示。頻繁にチェックする5〜10件を登録すると朝チェックが効率化。

タグ付け

プロパティ詳細・GTM詳細・SC詳細の「🏷 タグ・メモ」セクションで、カンマ区切りで複数タグを設定。例:

契約中 EC A社 月次レポート対象

一覧画面の「タグ」フィルタで絞り込み可能。たとえば「契約中」だけ表示して一括AI分析をかけることもできます。

メモ

クライアント担当者・連絡履歴・特記事項などを記録。詳細ページから保存可。ホームの「⭐ ユーザータグ別件数」で集計確認。

運用例: 「契約中」「トライアル」「休眠」「セミナー素材」「業界:EC」「業界:メディア」のように2軸で運用すると、後で集計・抽出しやすくなります。

GA4の計測精度を上げるチェックリスト

ヘルススコアを上げる＝計測精度を上げる、です。スコア要素を1つずつ確認してください。

1. 計測の基礎確認（スコア30点分）

  • 直近7日でSessions/Events/Usersが1以上か？ ← 0なら計測停止
  • データストリームが1件以上あるか？
  • Webストリームの場合、Measurement IDが正しいか？
  • サイトに gtag.js または GTM の GA4 Config タグが入っているか？

2. キーイベント設定（スコア20点分）

  • 主要CV（フォーム完了、購入、資料DL、SNS共有等）がキーイベントに昇格されているか？
  • 3件以上のキーイベントが設定されているか？
  • キーイベント名が snake_case で統一されているか？
  • GA4予約イベント（purchase等）と独自イベント名が衝突していないか？

→ 「AIで分析」で業種・URL・top eventsから推奨キーイベント5件を提案させると効率的。

3. カスタムディメンション設計（スコア15点分）

  • CDが1〜30件の範囲か？（50件上限、過多は管理コスト増）
  • 命名規約（プレフィックス・スネークケース・スコープ表記）が一貫しているか？
  • 使われていないCD・テスト残骸のCDがないか？
  • スコープ（Event/User/Item/Session）が用途に合っているか？

4. eコマース計測（スコア10点分）

  • ECサイトなら purchase が直近30日で発生しているか？
  • view_item / add_to_cart / begin_checkout / purchase のファネルが揃っているか？
  • items配列・value・currencyが正しく送信されているか？

5. 権限と運用

  • 自分（または代理人）がAdministrator権限を持っているか？
  • 除外フィルタ（内部IP・ボット・特定UA）が設定されているか？
  • クロスドメイン計測が必要なサイトで設定されているか？

GTMの計測精度を上げるチェックリスト

1. 基本構成（スコア30点分）

  • コンテナが公開されているか？（バージョンIDあり）
  • 少なくとも1つ以上のタグが入っているか？
  • GA4 Measurement IDが設定されたコンテナか？

2. GA4設定の整合性（スコア10点分）

  • gaawc（GA4 Configurationタグ）が1つ存在するか？
  • Measurement IDが本番用と一致しているか？（テスト用が紛れていないか）
  • Enhanced Measurementが有効化されているか？

3. レガシータグ整理（スコア15点分）

  • UA系タグ（タグタイプ ua）が残っていないか？ ← 即削除推奨
  • 古いYahoo!タグマネージャ・Floodlightの旧仕様タグが残っていないか？
  • 「2023年7月以前に作成されて触られていないタグ」がないか？

→ GTMコンテナ詳細でAI分析を実行すると、UA系タグ・廃止対象タグを自動検出。

4. 整理度（スコア10点分）

  • paused状態のタグが10%以下に抑えられているか？
  • 命名規約（接頭辞 GA4/Meta/Yahoo/... + 動詞）が一貫しているか？
  • 同じ目的のタグが複数あって統合候補になっていないか？
  • 使われていないトリガー・変数が残っていないか？

5. パフォーマンス・セキュリティ

  • カスタムHTMLタグで document.write や innerHTML への直接挿入を使っていないか？
  • 個人情報・APIキーがタグやコード内に直書きされていないか？
大規模コンテナの整理術: 400件超のコンテナはAIに「タグ命名規約と整理案」を依頼すると、削除候補リスト＋統合案を1,000字程度のドキュメントで返してくれます。

Search Consoleの状態を改善するチェックリスト

1. 基本設定（スコア15点分）

  • サイトマップが登録されているか？（/sitemap.xml をSearch Consoleから送信）
  • sitemap内のURLにエラーがないか？（404・noindex等）
  • 所有者または FullUser 権限を持っているか？

2. 流入規模（スコア30点分）

  • 直近28日でClick・Impressionが発生しているか？
  • Impression>0でClick=0の場合、タイトル・メタが検索意図とミスマッチの可能性
  • Impression=0なら、インデックス未登録または検索クエリと完全に外れている

3. CTR・順位の妥当性（スコア20点分）

  • 全体CTRが1%以上か？（理想は3%以上）
  • 平均掲載順位が30位以内か？
  • Impression高×CTR低のクエリ＝改善余地大（タイトル・メタ最適化のチャンス）

4. クエリ・ページの多様性（スコア20点分）

  • Clickが発生しているクエリが20件以上あるか？
  • Clickが発生しているページが20件以上あるか？
  • 1〜2ページに依存していないか（リスク分散）

5. GA4連携（スコア10点分）

  • 同じドメインのGA4プロパティが存在するか？
  • SC側でGA4プロパティのリンクを設定しているか？（管理画面 → 設定 → 関連付け）

テーマ切替

ヘッダ右上の 🌓 ボタンでライト⇔ダーク切替。設定はCookieに保存され次回も維持。

進捗パネル

データ更新中はヘッダのステータスをクリック、または自動表示される右上パネルで詳細進捗を確認できます。残り時間（ETA）と処理速度がリアルタイム計算されるので、長時間更新の予測が立ちます。

クイック検索

/ キーを押すと検索ボックスにフォーカス。URL末尾に /search?q=キーワード でも横串検索可能（メニュー非表示中）。

月次レポート用フロー

クライアント向け月次レポートやセミナー素材を作る時の流れ。

1

プロパティ一覧をCSVエクスポート

右上の「⬇ CSV」ボタン。全件 or フィルタ結果のヘルススコア・アラート・タグ・メモ込みでExcelで読める形式。

2

クライアントタグでフィルタ → 一括AI分析

例: タグ「契約中」「業界:EC」で絞って10件を一括AI分析。1〜2分で全件のサマリー＋課題＋アクションが揃う。

3

「TOP20課題」を活用

一括ジョブ詳細ページ上部の「重要度の高い課題TOP20」を確認。クライアント横断の傾向（CD命名揺れが多い／UA残存が多い等）が見えます。

4

各クライアントごとのアクションプランをコピー

個別プロパティ詳細から「📋 コピー」でAI生成のアクションプランをMarkdownにペースト。月次レポートのドラフトに使う。
