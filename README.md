# Analytics Inventory（アナリティクス・インベントリ）

複数のクライアントのGoogle Analytics 4・Google Tag Manager・Search Consoleを **一画面でまとめて管理** し、問題のあるサイトを発見してAIで改善案まで出せるツールです。

---

## 📖 このツールは何ができるの？

ウェブ解析コンサル・社内アナリストなど、**たくさんのGoogleアカウントで複数のGA4・GTM・Search Consoleを使っている人**向けの管理ツールです。

### こんなことが起きていませんか？

- 「あれ、Aクライアントのコンバージョン、ちゃんと計測できてたっけ？」と毎月確認が必要
- 「全クライアントのGA4設定を一覧で見られる仕組みがほしい」
- 「古いUniversal Analyticsのタグが残っているコンテナを見つけたい」
- 「ヘルスチェックを毎月やるのが大変なので自動化したい」
- 「AIにGA4の問題を診断してもらいたいが、毎回プロンプトを書くのが面倒」

このツールは、これらを **3日に1回自動で全アカウントを巡回し、問題を機械的に検出 → AIに改善案を出させる** ことで解決します。

---

## 🎯 主な機能

### 1. 一覧管理：全アカウントを横断して見える化

複数Gmailで管理している以下を **1つの画面に集約** します：

- **GA4プロパティ** — プロパティ名・計測状況・キーイベント数・eコマース計測有無・カスタムディメンション数・自分の権限
- **GTMコンテナ** — タグ数・トリガー数・変数数・紐づくGA4 Measurement ID・公開バージョン
- **Search Consoleサイト** — 直近28日のクリック数・表示回数・CTR・平均掲載順位・サイトマップ状況・トップクエリ

### 2. ヘルススコアリング：100点満点で健康度を可視化

各プロパティ・コンテナ・サイトを **100点満点でスコア化** し、A〜Fのグレードで評価します。

たとえばGA4プロパティの場合：
- 計測中であれば +30点
- キーイベント3件以上で +20点
- eコマース計測ありで +10点
- データストリーム設定済みで +10点
- ... など合計で算出

スコアが低いものから対応すればよい、という優先順位がつきます。

### 3. 問題の自動検出

機械的にチェックして、対応が必要な項目を **「今日の要対応」** としてホーム画面に表示します。

| 種別 | 検出される問題 |
|---|---|
| 📊 GA4 | 計測停止 / ストリーム未設定 / API取得エラー / キーイベント未設定 / カスタムディメンション過多 |
| 🏷 GTM | タグ0件 / 古いUAタグが残っている / GA4と連携できていない |
| 🔍 Search Console | クリック0 / サイトマップ未登録 / CTRが極端に低い |
| 🔗 クロス | Measurement ID重複 / GA4↔GTMの紐付け不整合 |

クリックすると **対象プロパティの一覧** と **対策方法のステップバイステップ** がモーダルで表示されます。

### 4. AIによる診断と改善案の提示

各プロパティ・コンテナをClaude（生成AI）に分析させると、以下を自動で返します：

- **所見**（2〜3行の総合判定）
- **課題リスト**（最大8件、severity high/medium/lowで色分け）
- **アクションプラン**（優先度付き最大6件、所要時間目安付き）
- **委任プロンプト**（外部AIに作業を依頼するための完成済みプロンプト）

「委任プロンプト」がポイントで、たとえば「GTMタグ実装のJSON設定ファイルを作って」のようなプロンプトを **AIが自動生成** してくれます。それをそのまま Claude.ai や ChatGPT に貼り付けると、具体的な実装ファイルが返ってきます。

### 5. 一括AI分析・自動診断

- **一括AI分析**：複数プロパティを選んで、まとめて分析（実行前にコストと所要時間を表示）
- **自動診断**：3日に1回のデータ更新後、新規に問題が発生したプロパティを最大15件、自動でAI分析

### 6. クライアント別ダッシュボード＋PDFヘルスレポート

GA4プロパティ・GTMコンテナ・SCサイトを「A社」「B社」のようにクライアント単位でグルーピングし、**クライアントごとの健康状態を1画面で確認**できます。ワンクリックで **PDFヘルスレポート**（スコア一覧・直近30日の変化・要対応の詳細）を出力でき、そのまま月次報告に添付できます。日本語フォント（IPAexゴシック）同梱なので、Windows/Mac/Linuxどこでも同じPDFが出ます。

個別の管理機能も：

- **お気に入り（★）** — よく見るプロパティを固定表示
- **タグ付け** — 「契約中」「EC」「A社」など自由なラベル
- **メモ** — クライアント担当者や経緯を記録
- **監視除外・スヌーズ（🔕）** — テスト用プロパティ等をアラート対象から外す。「30日だけ停止」の期限付きスヌーズも可能

### 7. 履歴・差分検知

データ更新のたびにスナップショットを保存し、**前回比でセッション急減・キーイベント変更・計測停止** を自動検出。「先週はあったのに今週なくなった」が一目で分かります。

### 8. 変化ログ（重要度付き自動監視）

差分検知をさらに一歩進め、スキャンごとの変化を **重要度付きイベント**（重大 / 注意 / 情報）として自動記録します。

- **重大** — 計測停止、キーイベント削除、データストリーム減少、GTMタグ全消失、GA4 Measurement IDの取り外し
- **注意** — セッション・検索クリックの急減（前回比-50%超）、タグ数減少、sitemapエラー増加
- **情報** — プロパティ/コンテナの追加、キーイベント追加、GTM新バージョン公開

「クライアントから数値がおかしいと連絡が来た」ときに、**いつ・何が変わったか** を数秒で切り分けられます。

急減とみなす減少率（既定 -50%）やCD過多・UA残存の警告件数などの**しきい値は設定画面（分析メニュー→監視しきい値設定）から変更可能**です。サイト規模に合わせて調整してください。

また、新しいバージョンが公開されるとホーム画面に更新バナーが表示されます（GitHubを1日1回チェック。更新はZIPを上書きするだけ、`data/`・`tokens/`・`.env` はそのまま引き継がれます）。

### 9. 週次通知（メール / Slack / Webhook）

毎週決まった曜日・時刻に、**KPIサマリー＋直近7日間の変化＋要対応の一覧（グレードD/F・エラー）** を受け取れます。通知先は3種類から選べ、**併用も可能**です：

- **メール** — HTMLメール。SMTP設定・プレビュー・テスト送信を画面から操作
- **Slack** — Incoming Webhookに、重要度絵文字付きのBlock Kitメッセージを投稿。チームで見るならこれが手軽
- **汎用Webhook** — サマリーを構造化JSONでPOST。Zapier / Make / Discord / 自社の受け口などに接続可能

Windowsのタスクスケジューラ連携用バッチ（`register_weekly_task.bat`）も同梱しており、サーバーを常時起動していなくても送信できます。

### 10. CSVエクスポート

クライアント報告書や社内共有用に、プロパティ・コンテナ一覧をCSVで出力。Excelで開いてそのまま使えます。

## 🛠️ セットアップの流れ

「Pythonって何？」レベルの方でも、以下の手順を順番に進めれば動かせます。専門用語の説明も含めます。

> 💡 **困ったら診断コマンド**: どのステップまで完了しているか分からなくなったら、ツールのフォルダで
> `python doctor.py`（または `run_doctor.bat` をダブルクリック）を実行してください。
> Python・ライブラリ・client_secret.json・認証・APIの有効化・データ収集を順に検査して、
> **NGの箇所と「次にやること」を日本語で表示**します。セットアップ中は何度実行してもOKです。

### ステップ1：Pythonをインストール（初回のみ）

Pythonとは、このツールを動かすためのプログラミング言語です。

1. https://www.python.org/downloads/ から最新版（3.10以上）をダウンロード
2. インストール時に **「Add Python to PATH」のチェックを必ずON** にする
3. インストール後、PowerShellを開いて `python --version` でバージョン確認

### ステップ2：このリポジトリをダウンロード

GitHub からこのコードを自分のPCにコピーします。

```powershell
cd C:\Users\<あなたのユーザー名>
git clone https://github.com/ogawa-cmyk/analytics-inventory.git
cd analytics-inventory
```

> 💡 gitが入っていない場合は GitHub の「Code」ボタン → 「Download ZIP」でも可

### ステップ3：依存ライブラリのインストール

ツールが使う外部ライブラリ（Google API用・Anthropic API用など）をまとめてインストールします。

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

これで `.venv` というフォルダにライブラリ群が入ります（システム全体を汚しません）。

### ステップ4：Google Cloud Console の設定（初回のみ・15分くらい）

Googleのプロパティやサイトのデータを取得するために、Googleに「このアプリにアクセス権を与えていいですか？」と認証する必要があります。

1. https://console.cloud.google.com/ にアクセス（Googleアカウントでログイン）
2. 上部のプロジェクト選択メニューから **新規プロジェクトを作成**（既存でも可）
3. 左メニュー「APIとサービス」→「ライブラリ」で、以下4つのAPIを **すべて有効化**：
   - Google Analytics Admin API
   - Google Analytics Data API
   - Tag Manager API
   - Search Console API
4. 左メニュー「OAuth同意画面」を開き：
   - User Type: **「外部」** を選択
   - アプリ名: `Analytics Inventory` などお好みで
   - **テストユーザー** に、使用したいGmailアドレスをすべて登録
5. 左メニュー「認証情報」→「認証情報を作成」→「OAuthクライアントID」
   - アプリの種類: **「デスクトップアプリ」**
   - 名前: `Analytics Inventory Desktop`
   - 作成後、JSONをダウンロード → `client_secret.json` として `analytics-inventory` フォルダ直下に保存

### ステップ5：Anthropic APIキーの取得（AI分析を使う場合）

AI分析機能には Anthropic 社の Claude API を使います。

1. https://console.anthropic.com/ でアカウント作成
2. 「API Keys」から新しいキーを発行
3. `analytics-inventory` フォルダ内に `.env` という名前のファイルを作成し、以下を1行だけ書き込み：
   ```
   ANTHROPIC_API_KEY=sk-ant-api03-あなたのキー
   ```

> 💰 コスト目安：1プロパティの分析が約12円（Sonnet 4.6使用時）。月の利用は数百円〜数千円程度。

### ステップ6：Google アカウントの認証（使うGmail分繰り返す）

PowerShellで以下を実行：

```powershell
python auth.py add
```

ブラウザが開いてGoogleログイン画面が出るので、認証したいGmailアドレスでログインして承認します。**使用したいGmailの数だけ繰り返し** 実行してください。

### ステップ7：初回データ収集

```powershell
python indexer.py
```

すべてのアカウントを順番に巡回し、データを収集します。**所要時間は20〜45分**（プロパティ数による）。

### ステップ8：UI起動

```powershell
python server.py
```

「Running on http://127.0.0.1:8788」と表示されたら、ブラウザで http://127.0.0.1:8788 を開きます。

🎉 これで使用開始です！

## 🔄 日々の運用

### 毎日（5分）

1. ブラウザで http://127.0.0.1:8788 を開く
2. **ホームの4列ヘルスサマリ** で全体状況をチェック
3. **「今日の要対応」タイル** に新しい問題が出ていないか確認
4. 気になるタイルをクリック → **対象と対策方法をモーダルで確認**
5. 必要に応じて該当プロパティを開いて対応

### 3日に1回（自動）

タスクスケジューラに登録すれば、3日ごとに **自動でデータ更新＋AI診断** が走ります（後述の自動起動設定参照）。

### 週次（30分）

1. 特定のクライアントのプロパティ詳細を開く
2. 「🤖 AIで分析・アクションプラン生成」を実行
3. AIが出した課題とアクションプランを確認
4. 委任プロンプトを使って外部AI（Claude.ai等）に具体的な実装案を作らせる
5. クライアントに共有

### 月次（クライアント報告書作成）

1. プロパティ一覧をCSVエクスポート
2. クライアントタグでフィルタ → 該当プロパティを一括AI分析
3. AI出力をMarkdownにコピーして報告書のドラフトに

## 🤖 自動起動・自動更新の設定（Windowsのみ）

### サーバを毎回手で起動するのが面倒な場合

`start_server_silent.bat` ファイルを、以下のフォルダにコピーするだけでログイン時に自動起動します：

```
C:\Users\<あなたのユーザー名>\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\
```

エクスプローラのアドレス欄に `shell:startup` と入力するとこのフォルダが開きます。

### 3日ごとの自動データ更新

PowerShellで以下を実行（一度だけ）：

```powershell
schtasks /Create /TN Analytics-Inventory-AutoRefresh ^
  /TR "C:\Users\<あなたのユーザー名>\analytics-inventory\run_indexer.bat" ^
  /SC DAILY /MO 3 /ST 03:00 /F
```

これで深夜3時に3日おきにデータが更新され、新規アラートのあるプロパティはAIが自動診断してくれます。

## 💡 よくある質問（FAQ）

### Q1. このツール、いくらかかりますか？
- ツール自体は **無料**
- Google API（GA4・GTM・SC）も **無料**
- Anthropic Claude API（AI分析を使う場合のみ）: 約12円/プロパティ（Sonnet 4.6）。月利用 数百〜数千円程度

### Q2. データはどこに保存されますか？
すべて **自分のPCのローカルディスク** にしか保存されません（`data/` フォルダ）。クラウドへのアップロードはありません。AI分析時はClaude APIに設定情報のみ送信され、実際の訪問者データは送られません。

### Q3. 他人と共有できますか？
- リポジトリは **Privateリポジトリ** なので、自分以外は見られません
- GitHubで Collaborator を追加すれば、信頼できる相手と共有可能
- ただし `.env`（APIキー）・`tokens/`（OAuth）・`data/`（顧客データ）は除外設定（gitignore）されているので、コラボレーターには各自セットアップが必要

### Q4. 「OAuth同意画面」って何？
Googleの仕組みで「このアプリにあなたのGoogleアカウントのデータを見せていいですか？」というユーザーへの確認画面のこと。テスト中は、登録した「テストユーザー」のGmailしか認証できない仕様です。

### Q5. Universal Analyticsのタグはなぜ削除推奨ですか？
2023年7月にGoogleが正式にUAの計測を終了しました。GTMにUAタグが残っていても発火しても無効値を返すだけですが、トリガー設定によっては予期しない動作を引き起こすリスクがあります。

### Q6. ヘルススコアが低い＝悪いサイト？
そうとは限りません。スコアは **計測実装の充実度** を表すもので、ビジネスの良し悪しではありません。低スコアは「計測設定の改善余地が大きい」という意味です。

### Q7. AI分析の「委任プロンプト」って何？
AIが「このアクション、外部のAIで作業できるなら、こういうプロンプトを投げてください」と用意してくれるテキストです。たとえば「GTM設定JSONを生成してください、現状はこれです...」のように、必要なコンテキストを全部含んだ完成版のプロンプトを1,000〜2,000字で出力します。それを Claude.ai や ChatGPT に貼り付けるだけで具体的な成果物が返ってきます。

### Q8. データ更新中に他の操作は可能？
はい。データ更新はバックグラウンドで走り、進捗バーがヘッダに表示されます。一覧閲覧・詳細閲覧・AI分析実行など他の操作は問題なくできます。

### Q9. 「ダークモード」はある？
あります。ヘッダ右上の 🌓 ボタンで切替可能。

### Q10. キーボードショートカット
- `g h` ホーム / `g p` GA4 / `g g` GTM / `g s` SC / `g a` 一括AI分析
- `?` ヘルプ / `/` 検索ボックスフォーカス

## 🛟 トラブルシューティング

### サーバが起動しない

- ポート8788が他で使われている可能性 → `stop_server.bat` でプロセス停止後、再度起動
- Pythonがインストールされていない → ステップ1から実施
- 仮想環境（`.venv`）が壊れている → 一度削除して再作成

### 「ImportError」とかいうエラーが出る

ライブラリのインストールが不完全。`pip install -r requirements.txt` を再実行。

### Search Consoleのデータが0件

Google Cloud ConsoleでSearch Console APIが有効化されていない可能性。SC一覧ページに表示される有効化リンクから設定してください。

### Gmail追加したのに認証エラーになる

OAuth同意画面の「テストユーザー」にそのGmailが登録されているか確認。未登録だとアクセス拒否されます。

### AI分析が失敗する

- `.env` にAPIキーが正しく書かれているか確認
- 失敗時は自動で1回リトライ → それでも失敗ならOpus 4.8（深い分析モデル）で再試行

## 🔧 技術スタック（エンジニア向け）

- **言語**: Python 3.10+
- **Webフレームワーク**: Flask 3.x
- **テンプレートエンジン**: Jinja2
- **フロントエンド**: Vanilla JS, CSS Variables（ダークモード対応）
- **APIクライアント**:
  - google-api-python-client（Analytics Admin / Data / Tag Manager / Search Console）
  - anthropic Python SDK（Claude API）
- **データ保存**: JSONファイル（軽量、人間可読、Git diff容易）
- **認証**: OAuth 2.0 デスクトップフロー
- **スケジューラ**: Windows タスクスケジューラ + スタートアップフォルダ

---

## 📝 ライセンス・配布について

このリポジトリは **個人および所属組織での利用** を想定したPrivateリポジトリです。

第三者への配布や公開を行う場合は、以下を必ず削除してください（gitignore済ですが念のため）：

- `.env`（APIキー）
- `client_secret.json`（OAuth設定）
- `tokens/`（OAuthトークン）
- `data/`（顧客データ全般）

## ❓ さらに詳しい使い方

サーバを起動した状態でブラウザのヘッダから：
- **使い方** (`/usage`) — 実運用フローの解説、シナリオ集、改善チェックリスト
- **ヘルプ** (`/help`) — 用語集、スコア計算の詳細、トラブルシューティング

それぞれ詳細な解説と画面イメージ付きで読めます。
